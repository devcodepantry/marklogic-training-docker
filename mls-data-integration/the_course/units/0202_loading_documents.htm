Loading Documents

Documents can be loaded into a MarkLogic database through a variety of ways, including:

  • MarkLogic Content Pump
  • Data Movement SDK
  • Java API or Node.js API
  • REST API
  • XQuery or JavaScript functions

MarkLogic Content Pump

To load documents through MarkLogic Content Pump, which is an open source, Java-based command line tool, JRE 1.8 or later needs to be installed, the MLCP files that correspond to the version of MarkLogic that you are using need to be downloaded from developer.marklogic.com/products/mlcp, and a port needs to exist for communicating with the database that you want to load documents into.

Linux example (using mlcp.sh):

~/Desktop/mlcp/bin/mlcp.sh import -host localhost -port 8055 -username admin -password admin -mode local -input_file_path ~/Desktop/mls-data-integration/unit02/data/accounting-system -output_uri_replace "/home/cent/Desktop/mls-data-integration/unit02/data/accounting-system/,'/accounting/'"

Windows example (using mlcp.bat):

c:\mlcp\bin\mlcp.bat import -mode local -host localhost -port 8055 -username admin -password admin -input_file_path C:\\mls-data-integration\\unit02\\data\\accounting-system -output_uri_replace "C:/mls-data-integration/unit02/data/accounting-system,'/accounting/'"

Reading a Document

To read a document, the URI of the document is used.

XQuery example:

fn:doc("/accounting/order-10072.json")

JavaScript example:

fn.doc("/accounting/order-10072.json")

REST API instance example:

curl --anyauth --user admin:admin -X GET "http://localhost:8055/v1/documents?uri=/accounting/order-10072.json"
