Understanding Template Driven Extraction

Template Driven Extraction (TDE) provides a way to dynamically generate rows or triples from documents, without modifying the documents.

TDE for Rows

Both SQL (Unit 6) and the Optic API (Unit 7) utilize templates for queries.

SQL, one of the many APIs supported by MarkLogic, provides an easy way for SQL-based reporting and visualization tools to access real-time row data from a MarkLogic database.

The Optic API, a fluent MarkLogic query API, provides a fluent way to make row, triple, and/or document queries from Java, JavaScript, XQuery, or REST.

TDE is done by creating a template, validating the template, testing the template against sample documents, and then inserting the template into a schemas database. Once the template is inserted into a schemas database, a view is created, from which rows are dynamically generated and then populated into the rows index for the database. Queries can then be run against the view, to provide real-time row results, even when the view includes joins or aggregates.

Documentation for TDE is in the Template Driven Extraction chapter of the Application Developer's Guide, at docs.marklogic.com/guide/app-dev/TDE.

TDE for Triples

As the rows index is based on the triple index, TDE can also be used to dynamically generate triples from data in documents, populated into the triples index, for semantic queries.

TDE for triples is useful when your documents contain some data that is naturally represented and queried as triples, such as for semantic queries. For example, to be able to run semantic queries related to products that a customer has purchased, triples could be used.

Documentation on TDE for triples is available in the Using a Template to Identify Triples in a Document chapter of the Semantics Developer’s Guide, at docs.marklogic.com/guide/semantics/tde.
