Installing the ODBC Driver

By installing the MarkLogic ODBC driver on a computer, any ODBC-based applications installed on the computer (e.g. Tableau or Cognos) then have the capability to talk to MarkLogic databases through ODBC (e.g. to send SQL queries and receive results).

Available for Linux and Windows, the MarkLogic ODBC driver can be downloaded from developer.marklogic.com/products/odbc.

The steps for installing the driver depend on the operating system of the computer that the driver is being installed on (i.e. Linux or Windows).

Linux

To install the MarkLogic 9 ODBC driver on a Linux computer, unixODBC (2.3.4 or later) is required, which can be downloaded from www.unixodbc.org.

To install the the MarkLogic ODBC driver package on Red Hat Linux, rpm can be used.

rpm -i mlsqlodbc-1.1-20140528.x86_64.rpm

To define a DSN (name representing the configuration for connecting to a specific MarkLogic database) in the current user's .odbc.ini file, the odbcinst command can be used.

odbcinst -i -s -f /opt/MarkLogic/templates/mlsql.template

Detail on the steps for installing the MarkLogic ODBC driver on a Linux computer are at docs.marklogic.com/guide/sql/odbc-driver#id_67222.

isql

In installing unixODBC, a tool called isql is also installed.

isql is handy for checking that an ODBC driver has been set up, and for testing the connection to a database.

isql -v MarkLogicSQL admin admin

In the example above, MarkLogicSQL represents the name of the ODBC data source, once it has been created.

Windows

The steps for installing the MarkLogic ODBC driver on a Windows computer are at docs.marklogic.com/guide/sql/odbc-driver#id_55653.

For Windows, there is a 32-bit and a 64-bit MarkLogic OBDC driver available. If needed, both can be installed (like if the 32-bit driver is needed by some applications, such as Excel, and the 64-bt driver is needed for others).
