Storing and Indexing Triples

Triples can be stored independently in a MarkLogic database (a.k.a. managed triples), or embedded within XML or JSON content documents (a.k.a. unmanaged triples).

Generally, if a triple is about content in a particular XML or JSON document, the best practice is to store the triple embedded in the document so that it inherits the permissions of the document, and so that, if the document is deleted, the triple is also deleted.

To be immediately available for queries, triples are automatically populated into the triple index during the loading process, regardless of whether they are embedded in a document or independent. The triple index is turned on by default, from MarkLogic 9 onwards.

Loading Triples

For embedded triples, the XML or JSON document is loaded as normal, using any of the document loading interfaces.

For loading independent triples (e.g. triples stored in the .rdf, .ttl, .n3, or .nt format), they can be loaded through any number of interfaces, including:

  • MLCP
  • REST API
  • Java API or Node.js API
  • XQuery or JavaScript functions

The format that independent triples come from, and the interface used for loading, do not affect how the independent triples are stored in MarkLogic. For more detail on loading independent triples, see the MarkLogic Semantics course, or the Semantics Developer's Guide at docs.marklogic.com/guide/semantics/loading#id_81259.

Lastly, along with being able to load triples independently, or embedded in a document, a third way to have triples is by using the templates feature (Unit 5) to populate the triple index with triples dynamically generated from data in existing documents. Detail on it is at docs.marklogic.com/guide/semantics/tde.
