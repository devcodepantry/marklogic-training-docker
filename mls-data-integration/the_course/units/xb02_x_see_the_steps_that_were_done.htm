Exercise: See the Steps That Were Done

In this exercise, you view the steps that were done to create the CentOS 7 virtual machine for this course. Realize that, if you go to reproduce these steps, some of the versions and URLs may have moved on.

Note: It is not necessary to build the virtual machine yourself. You can download it from MarkLogic University.

 

  1. Downloaded and installed VirtualBox (VirtualBox-5.1.6-110634-Win.exe).
  2. Downloaded CentOS ISO (CentOS-7-x86_64-DVD-1611.iso).

To create the virtual machine:

  1. In VirtualBox, clicked the New button.
  2. For Name, entered centos7_64.
  3. For Type, left Linux selected.
  4. For Version, left Red Hat (64 bit) selected.
  5. Clicked the Next button.
  6. For Memory size, entered 3072 (3 GB), and clicked the Next button.
  7. Left Create a virtual hard drive now selected, and clicked the Create button.
  8. Left VDI (VirtualBox Disk Image) selected, and clicked the Next button.
  9. Left Dynamically allocated selected, and clicked the Next button.
  10. For file name and location, left the default location of centos7_64 (C:\Users\rstryker\VirtualBox VMs\centos7_64).
  11. For size, entered 20 GB, and clicked the Create button.

To increase the performance of the virtual machine, specified 2 CPUs:

  1. In VirtualBox, with the virtual machine selected, clicked the System header.
  2. Selected the Processor tab.
  3. For Processor(s), ented 2.
  4. Clicked OK.
  5. Saw that the number of Processors listed is now 2.

To turn off transparent huge pages, set the I/O scheduler to deadline, and set huge pages to 3/8 system memory (virtual machine has 3 GB RAM, so 3072 * 0.375 = 1152, and then 1152 / 2.048 = 562.50 huge pages):

  1. Selected Applications - Utilities - Terminal to open a terminal window.
  2. To become root, entered su.
  3. For password, entered centos.
  4. Entered vim /etc/default/grub.
  5. Pressed the i key to be able to edit the file.
  6. At the end of GRUB_CMDLINE_LINUX, after quiet, entered:

     transparent_hugepage=never elevator=deadline
  7. Pressed the Esc key.
  8. Typed :wq and then pressed the Enter key to exit the file.
  9. To regenerate the boot configuration, entered:

    grub2-mkconfig –o /boot/grub2/grub.cfg
  10. To be able to set the number of huge pages to 3/8 of the system memory, entered vim /etc/sysctl.conf.
  11. Pressed the i key to be able to edit the file.
  12. Scrolled to the bottom of the file.
  13. Added vm.nr_hugepages=562.
  14. Pressed the Esc key.
  15. Typed :wq and then pressed the Enter key to exit the file.

  1. Opened Firefox and went to the Open menu (three bars icon in upper right) - Preferences - Privacy - remove individual cookies - Remove All.
  2. Clicked the Close button.
  3. Clicked clear your recent history.
  4. For Time range to clear, selected Everything.
  5. Clicked the Clear Now button.
  6. Closed all tabs.

To create a clone of the centos7_64 virtual machine:

  1. Closed any open windows.
  2. Shut down the virtual machine, by clicking the triangle in the upper-right corner, clicking the Shut Down button, and clicking the Power Off button.
  3. In VirtualBox Manager, with the virtual machine selected, selected Machine - Clone....
  4. For new machine name, entered centos7_64_data_integration.
  5. Selected the checkbox for Reinitialize the MAC address of all network cards.
  6. Clicked the Next button.
  7. Left Full clone selected.
  8. Clicked the Clone button.
  9. Waited for the new virtual machine to be generated.

To generate an .ova file for distribution:

  1. Selected File - Export Appliance.
  2. With the cloned virtual machine highlighted (centos7_64_data_integration), clicked the Next button.
  3. For File, specified a location and a file name of centos7_64_data_integration.ova (e.g. C:\Users\rstryker\Desktop\centos7_64_data_integration.ova).
  4. Clicked the Next button.
  5. Clicked the Export button.
  6. Waited for the .ova file to be generated.
  7. Uploaded the .ova file for distribution.
