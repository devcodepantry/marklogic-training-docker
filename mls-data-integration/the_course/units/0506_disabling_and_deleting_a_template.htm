Disabling and Deleting a Template

To remove a template, it should first be disabled, and then deleted.

By disabling a template before deleting it, the related index entries are cleared, which frees disk space. Also, for any triples templates, disabling the template before deleting it ensures that the triples do not get included in semantic queries.

Disabling a Template

To disable a template, the <enabled> element can be used within the template definition. By having the <enabled> element set to false, the template is disabled, and any rows previously generated by it are removed from the rows index.

<template xmlns="http://marklogic.com/xdmp/tde">
  <enabled>false</enabled>
  <context>/order</context>
  ...
</template>

Deleting a Template

Once a template has been disabled, and enough time has passed to ensure that the reindexing related to the template has finished, the template can be deleted by removing it from the schemas database.

xdmp:document-delete("/templates/purchases-template.xml")
