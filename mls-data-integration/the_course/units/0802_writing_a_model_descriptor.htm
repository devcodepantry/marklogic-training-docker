Writing a Model Descriptor

A model descriptor is for defining entity types, their attributes, and relationships between entities. To create a model descriptor, XML or JSON is used. For example, imagine wanting to create a model that defines entity types for products, orders, and customers.

{
  "info": {
    "title": "procycling",
    "version": "0.0.1",
    "baseUri": "http://procycling.com/entity-types",
    "description":"ProCycling schema to define entity types for products, orders, and customers."
  },
  "definitions": {
    // definition for product entity type
    // definition for order entity type
    // definition for customer entity type
  }
}

The descriptor must contain the title and version properties in the info section, and define at least one entity type with at least one property in the definitions section.

Once a model descriptor is deployed, the model can be used to generate code for entity manipulation tasks.

JSON most closely aligns with the internal representation of the model. If using XML to write the model descriptor, then es:model-from-xml() and es:model-validate() are needed to translate the descriptor when using functions that receive the model as input.

The modeling vocabulary was inspired by Swagger and the OpenAPI Specification.

Detail on creating model descriptors is available in the Creating and Managing Models section of the Entity Services Developer's Guide at docs.marklogic.com/guide/entity-services/models.
