Loading Records from a .csv File

MarkLogic Content Pump can be used to import rows from a .csv file to a MarkLogic database, with each row manifested as an XML or JSON document.

Source document:

MLCP command:

~/Desktop/mlcp/bin/mlcp.sh import -host localhost -port 8055 -username admin -password admin -mode local -input_file_path ~/Desktop/mls-data-integration/unit02/data/support-system/tech-support-calls_20170123.csv -input_file_type delimited_text -output_uri_prefix /tech-support/ -uri_id id -output_uri_suffix .json -document_type json -output_collections "raw,from_csv"

In the command above, the -input_file_type delimited_text option is used to state that the source document is to be read as a delimited text file.

The -document_type json option specifies that, for each row in the .csv source document, a .json file is to be created.

The -uri_id id option uses the value in the id column of the .csv file for the URI of the new document. If this option is not specified, the value of the first column is used by default.

To specify column values as a data type other than string, the -data_type option can be used.

Detail on all of these MLCP options is at docs.marklogic.com/guide/mlcp/import#id_70366.

In running the MLCP command, each row from the .csv file becomes a JSON document in the database.
