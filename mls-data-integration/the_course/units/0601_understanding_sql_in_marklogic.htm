Understanding SQL in MarkLogic

One of the many languages supported in MarkLogic is SQL.

MarkLogic supports SQL so that data stored in a MarkLogic database can be easily consumed by technologies that require a SQL data source (e.g. many business intelligence and reporting tools).

Like all of the MarkLogic interfaces, MarkLogic's SQL interface is able to serve data from MarkLogic in real-time, even if the database is quite large, due to the sophisticated indexes that are core to MarkLogic.

View Required

Once a template view has been created, SQL queries can be run against the view, to retrieve row data from the database.

Using SQL

SQL queries can be sent via server-side application code or through ODBC.

MarkLogic interfaces available for executing SQL queries against views include running a SQL query in Query Console, from XQuery using the xdmp:sql() function, from JavaScript using the xdmp.sql() function, or through the MarkLogic ODBC Driver.

Detail on using SQL with MarkLogic is available at docs.marklogic.com/guide/sql.


In MarkLogic 8 and earlier, range views is how views were created. Template views, available with MarkLogic 9 and onwards, are easier to create, and are faster, utilizing the rows index introduced in MarkLogic 9.

SQL-92

In general, MarkLogic supports the SQL-92 standard for queries, which includes joins and aggregates.

Detail on MarkLogic's SQL-92 support is at docs.marklogic.com/guide/sql/SQLqueries.
