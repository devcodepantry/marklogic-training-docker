Learning About MarkLogic as a Multi-Model Database

MarkLogic is a multi-model database, which supports modeling and storing data as documents or as triples, or a combination of both. In addition, as an enterprise NoSQL database, MarkLogic provides scalability, high availability, ACID transactions, certified security, built-in search, and optional schema.

The types of content that can be stored in a MarkLogic database are JSON documents, XML documents, triples, text files, and binary files (which includes most any type of document, such as images, videos, sound files, PDF documents, Word files, Excel files, PowerPoint files, etc.).

Data in Documents

In storing data as JSON or XML documents, data can be stored in its natural state, with the additional benefits of human-readability, ability to express hierarchy, superior support of sparse data, and the ability to use self-describing (JSON) property names or (XML) element names.

Most data is naturally modeled as a document. For example, sales orders, insurance claims, patient records, emails, employees, insurance policies, news articles, shipping documents, stock trades, blogs, tweets, reports, publications, manuals, patents, prescriptions, and press releases are all naturally modeled as documents.

In MarkLogic, not only can documents be stored in their natural form, but also, data can be added to the documents, while preserving the source content. For example, imagine wanting to store metadata, such as provenance or lineage data, with the source document. Or adding triples data, for semantic queries.

Triples

Triples can be embedded in JSON or XML documents, or stored on their own, in triplestore documents. Modeling data through triples provides a way to store relationships between entities, as well the option to store facts.

Documents vs. Triples

Documents provide a natural way to store entities (e.g. customers, orders, insurance policies, patents, articles), and triples provides a natural way to store relationships (e.g. customer purchased product, company ownedBy otherCompany, customer foundedIn 2013, etc.). Having said that, data integration could be done solely with documents, solely with triples, or a combination of both.
