Querying Triples

There are a number of ways that triples can be queried in MarkLogic.

One way to query triples in MarkLogic is to use SPARQL, which is the W3C Recommendation (i.e. the standard) for querying semantic data.

For example, a SPARQL query could be run to return the IRIs for all of the Napa Biking Tours board members.

SELECT ?o
WHERE {
        <﻿http://procycling.com/resource/Napa_Biking_Tours>
        <http://procycling.com/ontology/boardMember>
        ?o
      }

Or, a SPARQL query could be run to return the IRIs for all board members for all companies (e.g. to gain insight for sales):

SELECT ?s ?o
WHERE {?s <http://procycling.com/ontology/boardMember> ?o}

Result:

Or, a SPARQL could be run to return the IRIs for variations of a company name.

SELECT ?s  
WHERE {?s <http://www.w3.org/2002/07/owl#sameAs>* <http://procycling.com/resource/Napa_Biking_Tours>}

Result:



SPARQL queries can be run via Query Console, or from application code written in XQuery, JavaScript, or Java, or through the REST API.

Triples can also be queried via the Optic API, covered in Unit 7 of this course.

Yet another way triples can be queried is by using the XQuery cts:triples() or JavaScript cts.triples() function.

In all cases, MarkLogic takes advantage of the underlying triple index to provide real-time results.
