Understanding Entity Services

Entity Services is a framework for modeling data in MarkLogic.

By using Entity Services, entities can be modeled, entity instances can be created, and artifacts can be generated, for use in application development.

Detail on Entity Services is available at docs.marklogic.com/guide/entity-services.

In addition, there is an Entity Services on-demand video tutorial as part of the MarkLogic 9 What's New self-paced video course at www.marklogic.com/training-courses/whats-new-marklogic-9.

Using Entity Services

At a high level, the steps for using Entity Services are:

  1. Create a model descriptor
  2. Define entities, properties, and relationships within the model descriptor
  3. Generate the model
  4. Generate a conversion function (for converting source documents to be entity instances)
  5. Convert source documents into entity instances (enveloping the source documents, to include canonical data defined by the model)
  6. Generate any other desired artifacts (e.g. templates, schemas, database index settings, Search API options)
