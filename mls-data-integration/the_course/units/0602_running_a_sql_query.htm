Running a SQL Query

Query Console is handy for testing SQL queries against a view.

SQL queries can be run in Query Console by selecting the Query Type of SQL.

With SQL selected, queries can then be written against a view (e.g. purchases).

Aggregating

Along with support for common SQL clauses like SELECT, FROM, ORDER BY, and WHERE, aggregate functions like SUM(), MAX(), and COUNT() can also be used.

For example, to return the price of the most expensive product ordered, MAX() could be used.

SELECT MAX(purchases.price)
FROM purchases

Result:

Or, to count the number of items ordered by product, SUM() and GROUP BY could be used.

SELECT purchases.product, SUM(purchases.quantity)
FROM purchases
GROUP BY purchases.product
